In Python 3, HTMLParser subclasses must implement error(), although this requirement doesn't appear to be documented.

In Python 2, HTMLParser implements error() by raising an exception, which we don't want to do.

In any event, this method is called only on very strange markup and our best strategy is to pretend it didn't happen and keep going.
