XMPP Server's Router.

A router connects the different components of the XMPP service and routes messages between them based on the given routing table.

Connected components are trusted to have correct addressing in the stanzas they offer for routing.

A route destination of L{None} adds a default route. Traffic for which no specific route exists, will be routed to this default route.

@since: 8.2 @ivar routes: Routes based on the host part of JIDs. Maps host names to the

L{EventDispatcher<utility.EventDispatcher>}s that should receive the traffic. A key of L{None} means the default route.

@type routes: C{dict}

Add a new route.
