It delays by 100ms so reactor has a chance to go through readers and writers before attending pending delayed calls, so do not set delay to zero.

def deferred_f_from_coro_f(coro_f): (source)

Converts a coroutine function into a function that returns a Deferred.

The coroutine function will be called at the time when the wrapper is called. Wrapper args will be passed to it. This is useful for callback chains, as callback functions are called with the previous callback result.

def deferred_to_future(d): (source)

.. versionadded:: 2.6.0

Return an :class:`asyncio.Future` object that wraps *d*.

When :ref:`using the asyncio reactor <install-asyncio>`, you cannot await on :class:`~twisted.internet.defer.Deferred` objects from :ref:`Scrapy callables defined as coroutines <coroutine-support>`, you can only await on ``Future`` objects. Wrapping ``Deferred`` objects into ``Future`` objects allows you to wait on them:

    class MySpider(Spider):
        ...
        async def parse(self, response):
            d = treq.get('https://example.com/additional')
            additional_response = await deferred_to_future(d)
