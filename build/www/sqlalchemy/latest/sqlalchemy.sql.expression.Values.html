Represent a VALUES construct that can be used as a FROM element in a statement.

The _expression.Values object is created from the _expression.values function.

def __init__(self, *columns, **kw): (source)

Construct a _expression.Values construct.

The column expressions and the actual data for _expression.Values are given in two separate steps. The constructor receives the column expressions typically as _expression.column constructs, and the data is then passed via the _expression.Values.data method as a list, which can be called multiple times to add more data, e.g.:

from sqlalchemy import column
from sqlalchemy import values

value_expr = values(
    column('id', Integer),
    column('name', String),
    name="my_values"
).data(
    [(1, 'name1'), (2, 'name2'), (3, 'name3')]
)
Parameters
*columnscolumn expressions, typically composed using _expression.column objects.
**kwUndocumented
namethe name for this VALUES construct. If omitted, the VALUES construct will be unnamed in a SQL expression. Different backends may have different requirements here.
literal​_bindsDefaults to False. Whether or not to render the data values inline in the SQL output, rather than using bound parameters.
